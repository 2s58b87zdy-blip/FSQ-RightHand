FSQ COMMAND v8.1.1 HOTFIX - KLAR TIL GITHUB

1. Pak ZIP-filen ud.
2. Aabn GitHub repository: FSQ-RightHand.
3. Upload INDHOLDET af mappen direkte til repositoryets rod.
4. Overskriv eksisterende filer, hvis GitHub spoerger.
5. Commit direkte til main med teksten:
   FSQ Command v8.1.1 login hotfix
6. Aabn GitHub Actions og vent paa groent flueben.
7. Genstart App Service, hvis deployment ikke goer det automatisk.
8. Genindlaes FSQ Command med Ctrl+F5.

Azure App Settings:
- SQL_SERVER=atlas-command-sql.database.windows.net
- SQL_DATABASE=fsq-command
- AUTH_SECRET=<lang tilfaeldig tekst>
- INITIAL_OWNER_PASSWORD=<dit start-password>
- AZURE_STORAGE_CONNECTION_STRING=<eksisterende vaerdi>
- AZURE_STORAGE_CONTAINER=<eksisterende vaerdi>

Alternativt kan DATABASE_URL bruges i stedet for SQL_SERVER og SQL_DATABASE.
Managed Identity bruges automatisk, naar SQL_USER og SQL_PASSWORD ikke er angivet.

Vigtigt:
- GitHub Secret AZURE_WEBAPP_PUBLISH_PROFILE skal stadig eksistere.
- Upload ikke PublishSettings-filen til repositoryet.
- Hvis /api/auth/users stadig giver 500, vil den praecise fejl nu kunne ses i Azure Log Stream.
