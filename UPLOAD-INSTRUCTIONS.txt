FSQ COMMAND v9.2.1 - AZURE BLOB HOTFIX

1. Upload all files in this folder to the root of the GitHub repository and overwrite the existing files.
2. Wait for GitHub Actions to finish successfully.
3. Keep AZURE_STORAGE_CONTAINER set to: fsq-documents
4. Choose ONE Blob authentication method:

A. Existing connection string
   AZURE_STORAGE_CONNECTION_STRING must contain the COMPLETE value from:
   Azure Storage account > Security + networking > Access keys > Connection string.

B. Managed Identity (recommended)
   Add AZURE_STORAGE_ACCOUNT_URL, for example:
   https://YOUR-STORAGE-ACCOUNT.blob.core.windows.net

   On the Storage account, give the App Service identity fsq-right-hand this role:
   Storage Blob Data Contributor

5. Restart the App Service.
6. Sign in and open:
   /api/diagnostics/blob

A correct setup returns: "ok": true
